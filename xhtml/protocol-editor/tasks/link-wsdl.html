Linking with WSDL

The web services business protocol editor can take advantage of WSDL definition files to make sure that the messages used to describe the protocol match what the service actually supports.

  1. Click anywhere outside an element of the protocol to make the protocol the selected element. The properties view should reflect the change accordingly by showing only protocol properties.
  2. Change the protocol WSDL URL property to point to the service WSDL. For local files, please make sure that the URL starts with file://, for instance file://c/some-folder/my.wsdl or file:///home/julien/my.wsdl.
  3. When editing transitions, the message property is now displayed in a drop-down list containing the messages gathered from the WSDL description.
