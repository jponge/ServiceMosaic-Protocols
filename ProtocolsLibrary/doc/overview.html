Library overview

Purpose

This library offers a model of web services protocols for the Java™ platform. It contains the model interface and classes as well as persistance-related classes. Other functionnalities such as protocol operators will instead be part of another library project.

Organisation

The model is split into a set of interfaces contained in the fr.isima.ponge.wsprotocol package. Implementation classes are provided by the fr.isima.ponge.wsprotocol.impl package, but a user is expected to avoid using them directly as much as possible (the interfaces are preferred). Finally, the fr.isima.ponge.wsprotocol.xml package provides an XML persistance input/output manager. The writing is performed by a utility class while the reading is performed efficiently with SAX.

The instanciation of the model classes are expected to be made by using the well-known abstract factory design-pattern. To do that, an interface to such a factory is provided in the fr.isima.ponge.wsprotocol package. The class implementing the BusinessProtocol interface is responsible for protocol integrity.

Developping it

The sources archive adopts a fairly standard layout for a Java™ project.

  • src: contains the source code of the library
  • tests: contains the JUnit-based test suite
  • lib: contains the 3rd-party libraries required at runtime and/or for the tests suite
  • doc: contains documentation files such as this one.

The source code project can be used inside Eclipse as the .project and .classpath files are provided. Any change must be documented in Changes.txt and the test suite must be extended whenever it is possible.

Using it

The binaries archive simply contains the documentation (including Javadocs), the required runtime 3rd-party libraries as well as the library Jar file. This Jar can be reused in any project needing the WS-Protocol library. Please make sure that the library dependencies are met.

The library code uses a logging API and thus it is possible to gather some informations from it. To do that, just enable the DEBUG logging level.

Dependencies

The following libraries are needed by the WS-Protocol library.

  • Runtime:
    • Apache Log4j (optionnal, Commons Logging can plug another one in)
    • Apache Commons Logging
    • Apache Commons Logging API
  • Tests:
    • JUnit
    • Apache Commons JXPath

Ant build

There is a Ant build file (build.xml) that will by default package the application into both source and binary packages. It also builds the Javadocs.
